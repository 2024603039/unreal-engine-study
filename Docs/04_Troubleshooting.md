# 04. Troubleshooting

**작성일 : 2026-07-11**

---

# 개요

Unreal Engine 5.8과 Codex를 MCP로 연결하고 실습을 진행하는 과정에서 여러 문제가 발생하였다.

이번 문서에서는 실습하면서 직접 겪은 문제와 해결 방법을 정리하였다.

---

# 문제 1. Codex와 Unreal Engine 연결 실패

### 증상

Codex에서 MCP를 실행하였지만 Unreal Engine과 연결되지 않았다.

### 원인

Unreal Engine에서 MCP Server가 실행되지 않은 상태였다.

### 해결

Unreal Engine에서 MCP Server를 다시 실행한 후 Codex에서 다시 연결을 시도하였다.

그 결과 정상적으로 연결되는 것을 확인하였다.

---

# 문제 2. Tool 실행 권한 요청

### 증상

Codex가 Unreal Engine에서 작업을 수행할 때마다 권한을 허용하는 창이 나타났다.

### 원인

보안을 위해 사용자의 승인을 받아야 하는 기능이었다.

### 해결

**Allow for this session**을 선택하여 같은 작업을 반복할 때 권한을 다시 묻지 않도록 설정하였다.

---

# 문제 3. 레벨 저장 문제

### 증상

레벨을 저장할 때마다 이름을 입력하는 창이 계속 나타났다.

또한 프로젝트를 다시 열었을 때 작업한 내용이 보이지 않는 문제가 발생하였다.

### 원인

새로운 레벨(Untitled)을 만든 상태에서 작업을 진행하였지만 레벨을 정상적으로 저장하지 않은 상태였다.

### 해결

작업을 시작하기 전에 레벨을 먼저 저장하고, 이후에는 **Ctrl + S**를 이용하여 저장하는 방법을 익혔다.

---

# 문제 4. 생성한 Actor가 보이지 않음

### 증상

Cube를 생성했지만 화면에는 보이지 않았다.

### 원인

Actor가 카메라와 다른 위치에 생성되어 화면에서 보이지 않았다.

### 해결

Outliner에서 생성된 Actor를 선택한 뒤 **F 키**를 눌러 위치를 찾았고, 위치를 이동하여 정상적으로 확인하였다.

---

# 문제 5. Alt 복제가 되지 않음

### 증상

영상처럼 Alt 키를 누른 상태에서 바위를 복제하려고 했지만 복제되지 않았다.

### 원인

이동 축을 선택하지 않았거나 카메라 시점에서 작업하고 있었다.

### 해결

Move 도구를 선택한 후 이동 축을 드래그하면서 Alt 키를 눌러 정상적으로 복제하였다.

---

# 문제 6. 시퀀서에서 오브젝트 크기가 변경됨

### 증상

원기둥의 크기를 변경하였지만 카메라 시점으로 이동하면 다시 원래 크기로 돌아갔다.

### 원인

Sequencer가 Transform의 Scale 값을 관리하고 있었기 때문이다.

### 해결

Sequencer에서 Scale 키프레임을 확인하여 원인을 이해할 수 있었다.

---

# 문제 7. 프로젝트와 강의 화면이 다름

### 증상

강의 화면과 다르게 맵이나 폴더가 보이지 않았다.

### 원인

Blank 프로젝트와 강의에서 제공하는 Ignite Starter Pack 프로젝트가 서로 달랐다.

### 해결

강의에서 제공한 Starter Pack 프로젝트를 다운로드하여 같은 환경에서 실습을 진행하였다.

---

# 이번 실습을 통해 배운 점

처음에는 다양한 오류가 발생하여 어려움을 겪었지만 하나씩 원인을 찾아 해결하면서 언리얼 엔진의 사용 방법을 조금씩 이해할 수 있었다.

특히 레벨 저장 방식과 프로젝트 구조를 제대로 이해하는 것이 매우 중요하다는 것을 알게 되었다.

---

# 느낀 점

이번 실습에서는 단순히 기능을 사용하는 것뿐만 아니라 문제가 발생했을 때 원인을 찾고 해결하는 과정도 중요한 경험이었다.

앞으로도 언리얼 엔진을 공부하면서 새로운 문제가 발생하면 해결 방법을 함께 정리하여 비슷한 문제가 발생했을 때 참고할 수 있도록 기록할 계획이다.
