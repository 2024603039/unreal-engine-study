# 04. Troubleshooting

**작성일 : 2026-07-11**

---

# 개요

이번 실습에서는 Unreal Engine 5.8을 설치하고 Codex와 MCP를 연결하는 과정에서 여러 가지 문제가 발생하였다.

처음 접하는 프로그램이다 보니 설치부터 프로젝트 설정까지 어려운 부분이 많았지만, 하나씩 해결하면서 실습을 진행하였다.

이번 문서에서는 실습 중 직접 겪었던 문제와 해결 방법을 정리하였다.

---

# 문제 1. Unreal Engine 설치 시간이 너무 오래 걸림

### 증상

처음에는 Unreal Engine 5.8을 HDD에 설치하였다.

설치가 80% 이상에서 오랫동안 멈춰 있는 것처럼 보여 설치가 제대로 진행되고 있는지 알기 어려웠다.

결국 실습을 시작하기까지 예상보다 많은 시간이 걸렸다.

### 해결

기존 설치를 삭제한 후 SSD를 설치 위치로 변경하여 다시 설치하였다.

SSD로 다시 설치한 후에는 정상적으로 설치를 완료할 수 있었다.

### 배운 점

Unreal Engine은 용량이 매우 큰 프로그램이므로 SSD에 설치하는 것이 훨씬 효율적이라는 것을 알게 되었다.

---

# 문제 2. Codex와 Unreal Engine이 연결되지 않음

### 증상

Codex에서 MCP를 실행하였지만 Unreal Engine과 연결되지 않았다.

처음에는 설정을 잘못한 줄 알았다.

### 원인

MCP Server가 실행되지 않은 상태였다.

### 해결

Unreal Engine에서 MCP Server를 다시 실행한 후 Codex를 다시 연결하여 정상적으로 사용할 수 있었다.

---

# 문제 3. 강의 화면과 프로젝트가 달라 혼란스러움

### 증상

강의 화면에는 다양한 폴더와 맵이 있었지만 내 프로젝트에는 보이지 않았다.

처음에는 설치를 잘못한 줄 알고 여러 번 다시 확인하였다.

### 원인

Blank 프로젝트를 사용하고 있었고, 강의에서는 Starter Pack 프로젝트를 사용하고 있었다.

### 해결

강의에서 제공한 Starter Pack 프로젝트를 다시 다운로드하여 같은 환경에서 실습을 진행하였다.

---

# 문제 4. 생성한 Actor가 화면에 보이지 않음

### 증상

Cube를 생성했지만 화면에서는 보이지 않았다.

처음에는 생성이 실패한 줄 알았다.

### 원인

Actor가 카메라와 다른 위치에 생성되어 화면에서 보이지 않았다.

### 해결

Outliner에서 Actor를 선택한 뒤 **F 키**를 눌러 위치를 찾았고, 위치를 이동하여 정상적으로 확인할 수 있었다.

---

# 문제 5. Alt 키로 복제가 되지 않음

### 증상

영상처럼 Alt 키를 누르고 바위를 복제하려고 했지만 복제가 되지 않았다.

### 해결

Move 도구를 선택한 후 이동 축을 드래그하면서 Alt 키를 누르니 정상적으로 복제할 수 있었다.

---

# 문제 6. Sequencer에서 원기둥 크기가 다시 작아짐

### 증상

원기둥의 크기를 크게 변경했는데 카메라 시점으로 이동하면 다시 원래 크기로 돌아갔다.

처음에는 저장이 되지 않은 줄 알았다.

### 원인

Sequencer가 Transform 값을 관리하고 있었기 때문이다.

### 해결

Sequencer에서도 오브젝트의 위치와 크기를 제어할 수 있다는 것을 이해하게 되었다.

---

# 문제 7. 레벨 저장을 제대로 하지 않아 작업 내용이 사라짐

### 증상

몇 시간 동안 실습한 내용을 저장했다고 생각했는데 프로젝트를 다시 열어보니 작업했던 오브젝트들이 보이지 않았다.

처음에는 프로젝트가 손상된 줄 알고 많이 당황하였다.

### 원인

새로운 레벨(Untitled) 상태에서 작업을 계속 진행하였고, 레벨을 제대로 저장하지 않은 상태였다.

### 해결

레벨은 작업을 시작하기 전에 먼저 저장해야 한다는 것을 알게 되었다.

앞으로는 새로운 레벨을 만든 후 바로 저장하고, 작업 중에도 자주 저장하는 습관을 가지려고 한다.

---

# 이번 실습을 통해 배운 점

이번 실습에서는 설치부터 프로젝트 설정까지 생각보다 많은 시행착오를 겪었다.

특히 설치 시간이 오래 걸렸던 점과 레벨 저장을 제대로 하지 않아 작업한 내용을 잃어버린 경험이 가장 기억에 남았다.

하지만 하나씩 원인을 찾아 해결하는 과정을 통해 언리얼 엔진의 사용 방법을 조금씩 이해할 수 있었다.

---

# 느낀 점

처음에는 언리얼 엔진이 낯설어서 작은 문제도 해결하는 데 많은 시간이 걸렸다.

특히 레벨 저장을 제대로 하지 않아 작업한 내용을 잃어버렸을 때는 많이 아쉬웠지만, 그 경험을 통해 프로젝트와 레벨 관리의 중요성을 배울 수 있었다.

앞으로는 같은 실수를 반복하지 않도록 작업을 시작하기 전에 레벨을 먼저 저장하고, 중간중간 저장하는 습관을 가지려고 한다.

이번 실습은 어려운 점도 많았지만 그만큼 많은 것을 배울 수 있었던 시간이었다.
