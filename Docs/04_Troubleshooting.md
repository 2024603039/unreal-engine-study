# 04. Troubleshooting

**작성일 : 2026-07-11**

# 개요

이번 실습에서는 Unreal Engine 5.8을 설치하고 Codex와 MCP를 연결하는 과정에서 예상보다 많은 문제가 발생하였다.

처음에는 간단하게 설치하면 바로 실습을 시작할 수 있을 것이라고 생각했지만, 설치부터 프로젝트 설정, 레벨 저장까지 여러 문제를 직접 해결해야 했다.

이번 문서에서는 실습하면서 직접 겪었던 문제와 해결 과정을 정리하였다.

---

# 문제 1. Unreal Engine 설치 시간이 너무 오래 걸림

### 증상

처음에는 Unreal Engine 5.8을 HDD에 설치하였다.

설치가 시작된 후 진행률이 80% 이상에서 오랫동안 멈춰 있는 것처럼 보였고, 설치가 끝나지 않아 정상적으로 설치되고 있는 것인지 알기 어려웠다.

몇 시간 동안 기다려도 설치가 완료되지 않아 실습을 시작하지 못했다.

### 원인

Unreal Engine은 용량이 매우 큰 프로그램이기 때문에 HDD에서는 설치 속도가 많이 느릴 수 있었다.

### 해결

기존 설치를 삭제한 후 SSD를 설치 위치로 변경하여 다시 설치하였다.

SSD에 다시 설치한 후에는 정상적으로 설치가 완료되었고 Unreal Engine을 실행할 수 있었다.

### 배운 점

언리얼 엔진은 용량이 매우 크기 때문에 처음부터 SSD에 설치하는 것이 훨씬 효율적이라는 것을 알게 되었다.

---

# 문제 2. Codex와 Unreal Engine이 연결되지 않음

### 증상

Codex에서 Unreal MCP를 실행하였지만 Unreal Engine과 연결되지 않았다.

연결 오류가 발생하여 AI를 사용할 수 없었다.

### 원인

Unreal Engine에서 MCP Server가 실행되지 않은 상태였다.

### 해결

Unreal Engine에서 MCP Server를 다시 실행한 후 Codex에서 다시 연결을 시도하였다.

그 결과 정상적으로 연결되는 것을 확인하였다.

---

# 문제 3. Tool 실행 시 계속 권한 요청이 나타남

### 증상

Codex가 Unreal Engine에서 작업을 수행할 때마다 권한을 허용하는 창이 계속 나타났다.

같은 작업을 반복할 때마다 승인해야 해서 불편하였다.

### 원인

보안을 위해 사용자의 승인이 필요한 기능이었다.

### 해결

**Allow for this session**을 선택하여 같은 작업을 반복할 때 권한을 다시 요청하지 않도록 설정하였다.

---

# 문제 4. 생성한 Cube가 보이지 않음

### 증상

Cube를 생성하였지만 화면에서는 아무것도 보이지 않았다.

처음에는 생성이 실패한 줄 알았다.

### 원인

Actor는 정상적으로 생성되었지만 카메라와 다른 위치에 생성되어 화면에서 보이지 않았다.

### 해결

Outliner에서 Cube를 선택한 뒤 **F 키**를 눌러 위치를 찾았다.

그 후 위치를 이동하여 정상적으로 Cube를 확인할 수 있었다.

---

# 문제 5. Starter Pack 프로젝트와 Blank 프로젝트가 달라 혼란스러움

### 증상

강의 화면과 내 프로젝트 화면이 달라서 같은 실습을 따라 하기 어려웠다.

강의에서는 여러 폴더와 맵이 있었지만 내 프로젝트에는 아무것도 없어 처음에는 잘못 설치한 줄 알았다.

### 원인

나는 Blank 프로젝트를 사용하고 있었고, 강의에서는 Starter Pack 프로젝트를 사용하고 있었기 때문이다.

### 해결

강의에서 제공한 Starter Pack 프로젝트를 다운로드하여 같은 프로젝트에서 실습을 진행하였다.

---

# 문제 6. Alt 키로 바위 복제가 되지 않음

### 증상

강의처럼 Alt 키를 누른 상태에서 바위를 복제하려고 했지만 복제가 되지 않았다.

### 원인

Move 도구가 아닌 상태에서 작업하거나 이동 축을 선택하지 않은 상태였다.

### 해결

Move 도구를 선택한 후 이동 축을 드래그하면서 Alt 키를 눌러 정상적으로 복제할 수 있었다.

---

# 문제 7. Sequencer에서 원기둥 크기가 다시 작아짐

### 증상

원기둥의 크기를 크게 변경했는데 카메라 시점으로 이동하면 다시 원래 크기로 돌아갔다.

처음에는 저장이 되지 않은 줄 알았다.

### 원인

Sequencer가 Transform의 Scale 값을 관리하고 있었기 때문이다.

### 해결

Sequencer에서 Scale 키프레임이 적용되어 있다는 것을 확인하였고, 시퀀서가 오브젝트의 크기도 제어할 수 있다는 점을 이해할 수 있었다.

---

# 문제 8. 레벨 저장을 제대로 하지 않아 작업 내용이 사라짐

### 증상

몇 시간 동안 실습한 내용을 저장했다고 생각했는데 프로젝트를 다시 열어보니 작업한 바위와 원기둥 등이 모두 사라져 있었다.

처음에는 프로젝트가 손상된 줄 알고 매우 당황하였다.

### 원인

레벨을 저장하지 않은 상태(Untitled)에서 계속 작업을 진행하였다.

저장 버튼은 눌렀지만 새로운 레벨을 제대로 저장하지 않아 작업 내용이 맵에 저장되지 않았다.

### 해결

레벨은 작업을 시작하기 전에 먼저 저장해야 한다는 것을 알게 되었다.

앞으로는 새로운 레벨을 만든 후 바로 이름을 지정하여 저장하고, 작업 중에도 자주 저장하는 습관을 가지기로 하였다.

# 이번 실습을 통해 배운 점

이번 실습에서는 설치부터 실습까지 예상보다 많은 문제가 발생하였다.

특히 설치 시간이 오래 걸렸던 점과 레벨 저장을 제대로 하지 않아 작업한 내용을 잃어버린 경험이 가장 기억에 남았다.

처음에는 어려운 점이 많았지만 하나씩 원인을 찾아 해결하면서 언리얼 엔진의 사용 방법을 조금씩 이해할 수 있었다.

---

# 느낀 점

이번 실습에서는 언리얼 엔진을 처음 사용하다 보니 설치부터 프로젝트 관리까지 모든 과정이 낯설었다.

특히 몇 시간 동안 작업한 내용이 저장되지 않았다는 것을 알았을 때는 많이 당황했지만, 그 과정에서 레벨 저장의 중요성과 프로젝트 구조를 이해할 수 있었다.

앞으로는 작업을 시작하기 전에 레벨을 먼저 저장하고, 중간중간 저장을 자주 하면서 같은 실수를 반복하지 않도록 해야겠다고 생각하였다.

이번 실습은 어려운 점도 많았지만 그만큼 언리얼 엔진에 대해 많이 배울 수 있었던 시간이었다.
