# 04. Troubleshooting

작성일 : 2026-07-11

---

## 개요

Unreal Engine 5.8과 MCP를 설치하고 연결하는 과정에서 여러 가지 문제가 발생하였다.

처음에는 오류가 발생하면 많이 당황했지만, 하나씩 원인을 찾아 해결하면서 설치를 완료할 수 있었다.

이번 문서에서는 실습하면서 발생했던 문제와 해결 방법을 정리하였다.

---

## 1. Unreal Engine 설치가 오래 걸림

### 문제

처음에는 Unreal Engine을 HDD에 설치하였다.

설치 속도가 매우 느렸고, 설치가 중간에 멈추는 경우도 있었다.

### 해결

검색해 보니 HDD보다 SSD에 설치하는 것이 훨씬 빠르다는 것을 알게 되었다.

언리얼 엔진을 SSD에 다시 설치한 후 정상적으로 설치를 완료하였다.

### 배운 점

언리얼 엔진은 용량이 크기 때문에 가능하면 SSD에 설치하는 것이 좋다는 것을 알게 되었다.

---

## 2. MCP 연결이 되지 않음

### 문제

Codex와 Unreal Engine을 연결하려고 했지만 연결이 되지 않았다.

처음에는 설치를 잘못한 줄 알고 많이 헷갈렸다.

### 해결

MCP Server가 실행되지 않은 상태라는 것을 확인하였다.

서버를 다시 실행한 후 정상적으로 연결되었다.

### 배운 점

오류가 발생하면 바로 다시 설치하는 것보다 원인을 먼저 확인하는 것이 중요하다는 것을 알게 되었다.

---

## 3. 권한 승인 창이 계속 나타남

### 문제

Codex를 사용할 때마다 권한을 허용하는 창이 계속 나타났다.

### 해결

**Allow for this session**을 선택하여 같은 작업을 반복할 때 다시 묻지 않도록 설정하였다.

### 배운 점

보안을 위해 권한을 확인하는 과정이라는 것을 알게 되었다.

---

## 4. 저장 중 멈춤

### 문제

Level을 저장하는 과정에서 저장이 오래 걸리거나 프로그램이 멈춘 것처럼 보였다.

### 해결

잠시 기다려도 해결되지 않아 언리얼을 다시 실행한 후 다시 저장하였다.

이후에는 정상적으로 저장되었다.

### 배운 점

언리얼 엔진은 용량이 큰 프로젝트를 다루기 때문에 저장에도 시간이 걸릴 수 있다는 것을 알게 되었다.

---

## 5. Actor가 보이지 않음

### 문제

생성한 Cube가 화면에 보이지 않았다.

처음에는 생성이 안 된 줄 알았다.

### 해결

Outliner에서 Cube를 선택한 후 **F** 키를 눌러 위치를 확인하였다.

Cube가 너무 아래쪽에 생성되어 있었고, 위치를 수정하니 정상적으로 보였다.

### 배운 점

Actor가 안 보인다고 해서 생성되지 않은 것은 아니라는 것을 알게 되었다.

---

## 6. 이번 실습에서 느낀 점

이번 실습에서는 기능을 배우는 것보다 설치와 오류를 해결하는 시간이 더 길었다.

처음 접하는 프로그램이라 어려운 점도 많았지만, 하나씩 해결하면서 언리얼 엔진 사용 방법을 조금씩 이해할 수 있었다.

앞으로도 오류가 발생하면 당황하지 않고 원인을 먼저 찾아보는 습관을 가져야겠다고 생각하였다.
