# 02. Unreal Engine 5.8 MCP Setup

**작성일 : 2026-07-11**

---

# 1. 실습 목적

이번 실습에서는 Unreal Engine 5.8과 Codex를 MCP(Model Context Protocol)를 이용하여 연결하는 환경을 구축하였다.

또한 AI가 Unreal Engine과 정상적으로 연결되는지 확인하고, 이후 실습을 진행하기 위한 개발 환경을 준비하는 것을 목표로 하였다.

---

# 2. 개발 환경

| 항목 | 내용 |
|------|------|
| Unreal Engine | 5.8 |
| AI | OpenAI Codex (GPT-5.5) |
| 운영체제 | Windows |
| MCP | Unreal Engine Model Context Protocol |

---

# 3. Unreal Engine 설치

Epic Games Launcher를 이용하여 Unreal Engine 5.8을 설치하였다.

처음에는 HDD에 설치를 진행하였는데 설치 속도가 매우 느렸고, 진행률이 오랫동안 멈춰 있는 것처럼 보여 실습을 시작하는 데 많은 시간이 걸렸다.

이후 SSD로 설치 위치를 변경하여 다시 설치하였고 정상적으로 설치를 완료할 수 있었다.

이번 경험을 통해 Unreal Engine은 SSD에 설치하는 것이 훨씬 효율적이라는 것을 알게 되었다.

---

# 4. Codex 설치 및 실행

PowerShell에서 프로젝트 폴더로 이동한 후 Codex를 실행하였다.

처음에는 명령어 사용이 익숙하지 않아 어려웠지만 여러 번 실행해 보면서 사용 방법을 익힐 수 있었다.

또한 `/mcp` 명령을 이용하여 Unreal Engine과 연결되는지도 확인하였다.

---

# 5. MCP 연결

Unreal Engine에서 MCP Server를 실행한 후 Codex와 연결을 진행하였다.

처음에는 연결되지 않아 오류가 발생하였지만 MCP Server를 다시 실행한 뒤 정상적으로 연결할 수 있었다.

연결이 완료된 후에는 Unreal Engine과 Codex가 서로 통신할 수 있는 상태가 되었다.

---

# 6. Tool 확인

MCP가 정상적으로 연결된 후 사용할 수 있는 Tool을 확인하였다.

Tool을 통해 Unreal Engine 내부의 정보를 확인하거나 다양한 작업을 수행할 수 있다는 것을 알게 되었다.

아직 모든 기능을 사용해 보지는 않았지만 앞으로 실습하면서 하나씩 사용해 볼 예정이다.

---

# 7. 권한 설정

Codex가 Unreal Engine에서 작업을 수행할 때 권한을 허용하는 창이 나타났다.

실습에서는 **Allow for this session**을 선택하여 같은 작업을 반복할 때마다 권한을 다시 허용하지 않아도 되도록 설정하였다.

---

# 8. 실습 결과

이번 실습에서는 Unreal Engine 5.8과 Codex를 MCP를 이용하여 성공적으로 연결하였다.

또한 MCP Tool이 정상적으로 동작하는 것도 확인하였다.

이후에는 AI를 활용하여 Unreal Engine 프로젝트를 함께 작업할 수 있는 환경을 구축할 수 있었다.

---

# 9. 실습하면서 어려웠던 점

가장 어려웠던 부분은 설치 과정이었다.

처음에는 설치만 하면 바로 실습을 시작할 수 있을 것이라고 생각했지만 설치 시간이 예상보다 훨씬 오래 걸렸고, MCP를 연결하는 과정에서도 여러 오류가 발생하였다.

하지만 하나씩 원인을 찾아 해결하면서 환경을 구축할 수 있었고, 이후부터는 안정적으로 실습을 진행할 수 있었다.

---

# 10. 느낀 점

이번 실습을 통해 단순히 프로그램을 설치하는 것이 아니라 개발 환경을 직접 구축하는 과정도 매우 중요하다는 것을 알게 되었다.

처음에는 오류가 많이 발생하여 어려웠지만 하나씩 해결하면서 MCP가 어떻게 동작하는지 조금씩 이해할 수 있었다.

앞으로는 구축한 환경을 활용하여 언리얼 엔진의 그래픽스와 다양한 기능을 더 공부해 보고 싶다.
