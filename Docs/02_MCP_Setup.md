# 02. Unreal Engine 5.8 MCP Setup

작성일 : 2026-07-11

---

## 1. 실습 목표

이번 실습에서는 Unreal Engine 5.8과 Codex를 연결하기 위한 개발 환경을 구축하였다.

처음에는 설치만 하면 바로 사용할 수 있을 줄 알았지만, 여러 프로그램을 설치하고 연결하는 과정이 필요하다는 것을 알게 되었다.

---

## 2. 개발 환경

| 항목 | 내용 |
|------|------|
| Unreal Engine | 5.8 |
| AI | OpenAI Codex |
| 운영체제 | Windows 11 |
| MCP | Unreal Engine MCP |

---

## 3. Unreal Engine 설치

Epic Games Launcher를 이용하여 Unreal Engine 5.8을 설치하였다.

처음에는 HDD에 설치해서 속도가 너무 느렸고, 설치가 계속 멈추는 문제가 있었다.

그래서 SSD에 다시 설치하였고, 시간이 오래 걸렸지만 정상적으로 설치를 완료하였다.

이번 실습에서 가장 오래 걸렸던 부분이 설치 과정이었다.

---

## 4. MCP 연결

Unreal Engine과 Codex를 연결하기 위해 MCP를 설정하였다.

처음에는 연결이 바로 될 줄 알았지만 여러 번 오류가 발생하였다.


---

## 5. Codex 실행

PowerShell에서 프로젝트 폴더를 연 후 Codex를 실행하였다.

처음에는 명령어도 익숙하지 않았지만 여러 번 사용하면서 조금씩 익숙해졌다.

근데 사용하다보니 명령어도 입력도 가능하지만 "이 에셋을 더 실제 바다처럼 만들어줘." 와 같은 명령어도 가능하다는 것을 알게 되어 유용하게 쓸 것 같았다.

---

## 6. 연결 오류

처음에는 Codex와 Unreal Engine이 연결되지 않았다.

왜 안 되는지 몰라서 많이 당황했지만, 알고 보니 MCP 서버가 실행되지 않은 상태였다.

서버를 다시 실행하니 정상적으로 연결되었다.

이번 실습에서는 오류가 발생했을 때 원인을 하나씩 확인하는 것이 중요하다는 것을 배웠다.

---

## 7. Tool 연결 확인

연결이 완료된 후 `/mcp` 명령어를 사용하여 Unreal MCP가 정상적으로 연결되었는지 확인하였다.

Tool 목록이 정상적으로 나타나는 것을 확인하고 연결이 완료된 것을 알 수 있었다.

---

## 8. 어려웠던 점

이번 실습은 프로그램 설치가 가장 어려웠다.

특히 Unreal Engine 설치 시간이 매우 오래 걸렸고, MCP 연결 과정에서도 여러 오류가 발생하였다.

처음 접하는 내용이라 많이 어려웠지만, 하나씩 해결하면서 설치를 마칠 수 있었다.

---

## 9. 느낀 점

이번 실습은 실제 언리얼 프로젝트를 시작하기 전에 개발 환경을 만드는 과정이었다.

설치 과정은 오래 걸렸지만 앞으로 계속 사용할 환경이기 때문에 처음에 제대로 설치하는 것이 중요하다는 것을 알게 되었다.

앞으로는 MCP를 활용하여 언리얼 엔진을 더 효율적으로 사용하는 방법을 계속 학습할 예정이다.
