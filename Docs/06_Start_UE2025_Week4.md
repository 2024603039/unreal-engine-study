# 04. 시작해요 트윈모션 & 언리얼 엔진 2025 - 4주차

작성일 : 2026-07-16

---

## 1. 실습 목표

이번 실습에서는 Datasmith(강의에서 제공한 모델이 5.8 버전에 맞지 않아서 다른 모델로 대체했다.)로 가져온 모델을 Blueprint로 변환해서 기능을 추가할 수 있도록 하고, Blueprint 내부에서 여러 개의 부품들을 하나의 객체처럼 관리하는 방법을 학습하였다.

또한 Blueprint Editor의 구성과 부품 계층 구조를 이해하고, 개별 구성을 선택하여 수정하는 방법을 익혔다.

---

## 2. 실습 내용

### 2-1. 프로젝트 열기
프로젝트를 실행한 후 datasmith로 가져온 토스터 모델을 확인한 후,
아웃라이너에서 토스터 객체를 선택하고, 뷰포터에서 모델이 정상적으로 배치되어 있는지 확인했다.


### 2-2. Blueprint 생성

토스터 모델을 Blueprint로 변환한 후 Blueprint Editor를 열어 내부 구조를 확인하였다.

처음에는 그냥 토스터 하나인 줄 알았지만 Blueprint 안에서는 여러 개의 부품(Component)으로 나누어 관리되고 있다는 것을 알게 되었다.
다음과 같은 부품들이 포함되어 있었다.

- Body
- FrontInnerPanel
- BottomInnerPanel
- DoorHinge
- Screw
- Handle

각 부품은 독립적인 Static Mesh Component로 구성되어 있었으며, 하나의 Blueprint 안에서 계층 구조로 관리되고 있었다.

---

### 2-3. Blueprint Editor 확인

Blueprint Editor에서는 다음과 같은 창의 역할을 확인하였다.

- Components(토스터 구성 부품)
- Viewport(w,e,r 눌러보면서 원하는 크기, 방향 수정)
- Details(손잡이의 색과 재질 수정)

이번 실습에서는 Component 구조를 확인하고 각 부품을 선택하여 수정하는 연습을 진행하였다.

---

### 2-4. Component 선택

Blueprint 내부에서 원하는 부품을 선택하면 뷰포터에서 해당 부품이 강조되는 것을 확인하였다.

또한 Details 패널을 이용하여 선택한 Component의 정보를 확인해봤다.

---

## 3. 실습하면서 어려웠던 점

처음에는 같은 토스터 모델인데 왜 Blueprint를 열면 부품들이 따로 보이는지 이해하지 못했다.

여러 번 직접 선택해 보면서 하나의 모델도 여러 개의 부품으로 이루어져 있고, Blueprint에서는 그 부품들을 함께 관리한다는 것을 알게 되었다.
---
## 4. 이번 실습에서 배운 점

이번 실습을 통해 Blueprint가 단순히 스크립트를 작성하는 기능만 있는 것이 아니라 여러 개의 Mesh를 하나의 객체로 관리하는 역할도 수행한다는 것을 알게 되었다.

Blueprint 내부에서 Component를 수정하면 하나의 객체를 보다 효율적으로 관리할 수 있다는 점을 이해하였다.

그리고 나중에 사람 모델을 사용하면 표정도 똑같은 방식으로 수정할 수 있는지 궁금해졌다. 빨리 캐릭터로 해보고 싶다.


---

## 5. 느낀 점

이번 실습은 단순히 강의를 따라 하는 것보다 Blueprint의 구조를 직접 확인하고 각 Component를 선택해 보면서 이해하는 것이 훨씬 도움이 되었다.

아직 Blueprint 사용에는 익숙하지 않지만 반복해서 실습하면 언리얼 엔진의 객체 관리 방식에 점차 익숙해질 것이라고 생각한다.

앞으로도 실습 과정을 직접 정리하며 학습 내용을 꾸준히 기록할 계획이다.
