# 01. Unreal Engine 5.8 MCP Study

**작성일 : 2026-07-11**

---

# 1. 실습 목적

이번 실습에서는 Unreal Engine 5.8에서 사용할 수 있는 MCP(Model Context Protocol)를 직접 설치하고 Codex와 연결하는 과정을 진행하였다.

또한 AI와 언리얼 엔진을 함께 사용하는 방법을 익히고, 앞으로 그래픽스와 언리얼 엔진을 공부할 때 AI를 어떻게 활용할 수 있는지 알아보는 것을 목표로 하였다.

---

# 2. MCP란?

MCP(Model Context Protocol)는 AI와 프로그램을 연결해 주는 기능이다.

기존에는 AI가 질문에 답하거나 코드를 작성하는 역할을 했다면, MCP를 이용하면 AI가 Unreal Engine과 연결되어 프로젝트를 확인하거나 필요한 작업을 수행할 수 있다.

이번 실습을 통해 AI가 단순히 답변만 하는 것이 아니라 언리얼 엔진과 함께 사용할 수 있다는 것을 알게 되었다.

| 항목 | 내용 |

# 4. 실습 내용

이번 실습에서는 다음과 같은 내용을 진행하였다.

- Unreal Engine 5.8 설치
- Codex 설치 및 로그인
- MCP 연결
- Unreal Engine과 Codex 연동 확인
- Tool 목록 확인

처음에는 연결 과정이 어렵게 느껴졌지만 하나씩 설정하면서 정상적으로 사용할 수 있었다.

---

# 5. MCP를 사용해 보면서

MCP를 연결한 후 AI가 현재 열려 있는 프로젝트를 확인하고 필요한 정보를 가져올 수 있다는 것을 확인하였다.

또한 AI에게 작업을 요청하면 언리얼 엔진과 연결하여 필요한 작업을 수행할 수 있다는 점이 인상적이었다.

아직 모든 기능을 사용해 본 것은 아니지만 앞으로 다양한 실습에서 많이 활용할 수 있을 것이라고 생각하였다.

---

# 6. 그래픽스와의 연관성

교수님께서 말씀해 주신 것처럼 이번 실습에서 가장 중요한 것은 AI를 사용하는 것이 아니라 언리얼 엔진에서 그래픽 요소들이 어떻게 연결되는지 이해하는 것이라고 생각하였다.

앞으로는

- Mesh
- Material
- Lighting
- Camera
- Animation

등이 어떤 과정으로 연결되어 화면이 만들어지는지 중점적으로 공부할 예정이다.

---

# 7. 실습을 하면서 어려웠던 점

처음에는 Unreal Engine 설치부터 시간이 오래 걸렸다.

또한 MCP를 연결하는 과정에서도 오류가 발생하여 바로 실습을 진행하지 못하였다.

하지만 오류를 하나씩 해결하면서 언리얼 엔진의 구조와 MCP 연결 방법을 이해할 수 있었다.

---

# 8. 느낀 점

이번 실습은 처음 접하는 내용이라 어려운 부분이 많았지만 직접 설치하고 연결하는 과정을 경험하면서 AI와 Unreal Engine을 함께 사용할 수 있다는 점을 이해할 수 있었다.

앞으로는 언리얼 엔진의 그래픽스 기능을 공부하면서 AI를 활용하여 더 효율적으로 프로젝트를 진행해 보고 싶다.
