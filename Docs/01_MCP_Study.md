# 01. Unreal Engine 5.8 MCP Study

작성일 : 2026-07-11

---

## 1. 실습 목표

이번 실습에서는 Unreal Engine 5.8에서 사용할 수 있는 MCP(Model Context Protocol)가 무엇인지 알아보고, AI와 언리얼 엔진이 어떻게 연결되는지 학습하는 것을 목표로 하였다.

처음에는 MCP가 무엇인지 전혀 몰랐지만, 직접 설치하고 사용하면서 AI가 언리얼 프로젝트를 읽고 작업할 수 있도록 도와주는 기능이라는 것을 이해하는 것이 이번 실습의 목표였다.

---

## 2. MCP란?

MCP(Model Context Protocol)는 AI와 프로그램을 연결해 주는 기능이다.

기존에는 AI에게 질문만 할 수 있었다면, MCP를 사용하면 AI가 현재 열려 있는 언리얼 프로젝트를 확인하고 필요한 작업을 도와줄 수 있다.

처음에는 일반 ChatGPT와 어떤 차이가 있는지 잘 이해하지 못했지만, 실제로 언리얼 프로젝트와 연결되는 것을 보면서 차이를 알게 되었다.

언리얼엔지니어링에서 "색 바꿔줘" 이렇게 시키면 ai가 바꿔주는 것이다.

---

## 3. Unreal Engine에서 MCP 사용하기

Unreal Engine 5.8에서는 MCP를 이용하여 AI와 언리얼 엔진을 연결할 수 있다.

AI는 언리얼 프로젝트의 정보를 확인하거나 여러 기능을 사용할 수 있다.

예를 들어

- 현재 Level 확인
- Actor 정보 확인
- 프로젝트 상태 확인

등의 작업을 진행했다.

---

---

## 5. Toolset 확인

MCP에는 여러 Toolset이 있었다.

AI가 사용할 수 있는 기능들이 모여 있는 목록이라는 것을 알게 되었다.

---

## 6. 이번 실습에서 알게 된 점

이번 실습에서는 AI가 단순히 답변만 하는 것이 아니라 언리얼 프로젝트를 읽고 작업을 도와줄 수 있다는 것을 확인하였다.

기존에 사용하던 ChatGPT와는 조금 다른 방식이라는 점이 가장 신기했다.

시키는 걸 ai가 다해주니 이제는 진짜 ai를 잘 활용하는 사람이 잘될 것이라는 것을 실감했다.

---

## 7. 어려웠던 점

MCP라는 개념이 처음이라 용어도 어렵고 구조도 이해하기 쉽지 않았다.

특히 AI와 언리얼 엔진이 어떻게 연결되는지 처음에는 잘 이해하지 못했다.

하지만 직접 설치하고 연결하는 과정을 따라 하면서 조금씩 이해할 수 있었다.

---

## 8. 느낀 점

이번 실습은 새로운 기능을 배우는 것보다 MCP가 어떤 역할을 하는지 이해하는 시간이었던 것 같다.

아직은 어려운 부분도 많지만 앞으로 언리얼 실습을 계속 진행하면서 자연스럽게 익숙해질 것이라고 생각한다.

AI가 프로젝트를 확인하면서 작업을 도와준다는 점이 가장 신기했고, chatgpt를 많이 쓰듯이 앞으로 실습에서도 많이 활용해 보고 싶다.
