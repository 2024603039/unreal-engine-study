# 01. Unreal Engine 5.8 MCP Study

**작성일 : 2026-07-11**

---

# 1. 실습 목적

이번 실습에서는 Unreal Engine 5.8에서 사용할 수 있는 MCP(Model Context Protocol)가 무엇인지 알아보고, AI와 언리얼 엔진을 연결하는 방법을 공부하였다.

또한 Codex를 언리얼 엔진과 연결하여 AI를 활용한 개발 환경을 직접 구축해 보고, AI가 언리얼 엔진 작업을 어떻게 도와줄 수 있는지 확인하는 것을 목표로 하였다.

---

# 2. MCP란?

MCP(Model Context Protocol)는 AI와 프로그램을 연결해 주는 기능이다.

기존에는 AI가 코드만 작성하거나 질문에 답하는 역할을 했다면, MCP를 사용하면 AI가 언리얼 엔진과 연결되어 프로젝트의 정보를 확인하거나 필요한 작업을 수행할 수 있다.

이번 실습을 통해 AI가 단순히 답변만 하는 것이 아니라 언리얼 엔진과 함께 사용할 수 있다는 것을 알게 되었다.

---

# 3. Unreal Engine 5.8에서 MCP 사용

Unreal Engine 5.8에서는 MCP를 이용하여 AI와 언리얼 엔진을 연결할 수 있다.

이번 실습에서는 Codex를 연결한 후 MCP가 정상적으로 동작하는지 확인하였다.

MCP를 사용하면 AI가 다음과 같은 작업을 수행할 수 있다는 것을 확인하였다.

- 현재 프로젝트 확인
- 레벨 확인
- Actor 생성 및 이동
- Asset 검색
- Scene 정보 확인

---

# 4. MCP 동작 방식

이번 실습에서는 다음과 같은 순서로 MCP가 동작하였다.

```
사용자

↓

Codex

↓

MCP

↓

Unreal Engine
```

사용자가 명령을 입력하면 Codex가 이를 해석하고, MCP를 통해 Unreal Engine에 전달하여 작업을 수행하는 방식이다.

---

# 5. MCP Tool 확인

MCP에는 여러 가지 기능이 포함되어 있는 Tool이 있었다.

이번 실습에서는 Tool 목록을 확인하면서 어떤 기능을 사용할 수 있는지 살펴보았다.

대표적으로 다음과 같은 기능을 확인하였다.

- Scene 관련 기능
- Actor 관련 기능
- Material 관련 기능
- Animation 관련 기능
- Data Asset 관련 기능

아직 모든 기능을 사용해 보지는 않았지만, 앞으로 다양한 작업에 활용할 수 있을 것으로 생각하였다.

---

# 6. MCP의 장점

이번 실습을 진행하면서 다음과 같은 장점을 알게 되었다.

- AI와 Unreal Engine을 연결하여 사용할 수 있다.
- 반복되는 작업을 AI가 도와줄 수 있다.
- 프로젝트 정보를 AI가 확인할 수 있다.
- 여러 역할의 AI를 함께 사용할 수 있다.

---

# 7. 이번 실습 내용

이번 실습에서는 다음 내용을 수행하였다.

- Unreal Engine 5.8 설치
- Codex 설치 및 로그인
- Unreal MCP 연결
- MCP Tool 확인
- AI Agent Team 개념 확인
- AI를 이용한 기본 작업 실행
- GitHub 저장소 생성 및 문서 작성

---

# 8. 느낀 점

처음에는 MCP라는 개념이 어렵게 느껴졌지만, 직접 설치하고 Codex와 연결해 보면서 AI를 언리얼 엔진과 함께 사용할 수 있다는 것을 이해할 수 있었다.

아직 모든 기능을 사용해 본 것은 아니지만, 앞으로 언리얼 엔진을 공부하면서 AI를 활용하면 작업을 더 빠르고 편하게 진행할 수 있을 것이라고 생각하였다.

앞으로는 MCP의 다양한 기능을 더 익히고, 실제 프로젝트에도 활용해 보고 싶다.
